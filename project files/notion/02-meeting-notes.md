## جلسه ۱ — کیک‌اف پروژه (Kickoff)

**تاریخ:** یکشنبه ۱۲ خرداد ۱۴۰۴ (June 1, 2026)
**زمان:** ۱۸:۰۰ – ۲۰:۰۰
**مکان:** آنلاین (Discord)
**شرکت‌کنندگان:** امیرمسعود آشوری، مجیدرضا کریمی، محمد مجتبی کچویی

### دستور جلسه
۱. معرفی پروژه و اهداف
۲. انتخاب استک فنی اولیه
۳. تعریف وظایف و نقش‌ها
۴. برنامه‌ریزی Sprint 1

### بحث و تصمیمات

**موضوع ۱: ایده پروژه**
تصمیم گرفته شد یک chatbot تخصصی برای مهارت‌های بقا در بحران ساخته شود. دلیل انتخاب: نبود منبع فارسی جامع برای این موضوع، و ارزش عملی بالا. پروژه باید مبتنی بر هوش مصنوعی باشد و از متدولوژی Agile استفاده کند.

**موضوع ۲: استک فنی**
- **Backend:** FastAPI (Python) — سریع، async، مناسب ML
- **Vector DB:** Qdrant — تصمیم اولیه، نیاز به بررسی بیشتر
- **Embedder:** sentence-transformers (multilingual) — نیاز به تست
- **LLM:** Ollama + Qwen2.5 — اجرای محلی بدون نیاز به API key
- **Frontend:** HTML/JS ساده — بدون framework

**موضوع ۳: تقسیم نقش‌ها**
- امیرمسعود آشوری: مدیریت پروژه + توسعه ارشد (معماری، backend، frontend)
- مجیدرضا کریمی: مهندسی هوش مصنوعی (RAG pipeline، embedder، reranker)
- محمد مجتبی کچویی: توسعه‌دهنده backend (API، ادغام سرویس‌ها، تست)

**موضوع ۴: محتوای پایگاه دانش**
فهرست اولیه موضوعات بحران تهیه شد: زلزله، سیل، آتش‌سوزی، CPR، خونریزی، سوختگی، شکستگی، خفگی، کیت اضطراری. تصمیم: موضوعات بیشتری در طول Sprint 1 اضافه شود.

| اقدام | مسئول | مهلت |
|-------|--------|------|

### بحث و تصمیمات

**موضوع ۱: وضعیت پایگاه دانش**
تا این لحظه ۸ فایل MD نوشته شده بود. کیفیت محتوا خوب بود اما زمان نوشتن بیشتر از انتظار طول کشید. تصمیم: از AI برای تسریع نوشتن محتوای فارسی استفاده شود (با بررسی دقیق و ویرایش توسط تیم).

**موضوع ۲: مشکل embedder اولیه**
مجیدرضا گزارش داد که `paraphrase-multilingual-mpnet-base-v2` در تست با سوالات فارسی دقت پایینی داشت. سوالات فارسی گاهی به passage‌های نامرتبط map می‌شدند. تصمیم: تحقیق روی `intfloat/multilingual-e5-base` که در leaderboard‌های مشابه نتایج بهتری داشت.

**موضوع ۳: اضافه کردن موضوعات تخصصی‌تر**
تصمیم گرفته شد موضوعاتی مثل جنگ/درگیری مسلحانه، تشعشعات هسته‌ای، حوادث شیمیایی/بیولوژیکی نیز پوشش داده شوند تا پروژه جامع‌تر باشد.

## جلسه ۵ — بررسی میانی اسپرینت ۲

**تاریخ:** چهارشنبه ۲۶ خرداد ۱۴۰۴ (June 15, 2026)
**زمان:** ۲۱:۰۰ – ۲۲:۳۰
**مکان:** آنلاین (Discord)
**شرکت‌کنندگان:** همه اعضای تیم

### بحث و تصمیمات

**مشکل اصلی: سوالات خارج از دامنه**
محمد مجتبی در تست مشاهده کرد که سوالاتی مثل «رستوران خوب تهران؟» هم پاسخ می‌گیرند. این مشکل اساسی است چون chatbot باید فقط در حوزه بحران پاسخ دهد.

**راه‌حل‌های پیشنهادی:**
۱. threshold روی vector similarity score
۲. افزودن reranker + threshold روی rerank score

مجیدرضا پیشنهاد داد که reranker بهتر است چون cross-encoder رابطه معنایی سوال-پاسخ را واقعی‌تر ارزیابی می‌کند.

**تصمیم:** `BAAI/bge-reranker-v2-m3` اضافه می‌شود. threshold در config قرار می‌گیرد تا قابل تنظیم باشد.

**وضعیت سایر تیکت‌ها:**
- API endpoint کار می‌کند (HB-14 ✅)
- Chunking نهایی شد (HB-11 ✅)
- Ollama متصل شد (HB-15 ✅)
- UI ابتدایی آماده (HB-17 ~80%)

### اقدامات

| اقدام | مسئول | مهلت |
|-------|--------|------|
| افزودن reranker به پایپ‌لاین | مجیدرضا | ۲۹ خرداد |
| تست off-topic filtering | محمد مجتبی | ۲۹ خرداد |
| بهبود UI و RTL styling | امیرمسعود | ۳۰ خرداد |
